# ACS 资料采集助手 — UI 重构方案

---

## 1. 产品新定位

| 原定位 | 新定位 |
| --- | --- |
| ACS 业务控制台 | **ACS 资料采集助手** |
| 运维后台风格 | **桌面资料工具风格** |

**一句话：导入网址或文件，自动采集、清洗、预览并导出资料。**

---

## 2. 用户主流程

```
新建采集任务 → 导入资料来源 → 选择字段规则 → 开始采集 → 表格预览结果 → 导出 Excel/JSON/Markdown
```

---

## 11. 合规边界位置

不再全屏展示安全警告。改为：

1. **新建采集页**底部小字："请确保资料来源为公开授权页面。"
2. **高级设置 > 合规检查**：完整安全边界列表（折叠面板）。
3. **遇到 403/401**：在采集结果表格标注"❌ 无法访问"，不重试，不绕过。

---

## 12. 与现有 ACS 模块对应

| 新 UI 页面 | 原有 ACS 模块 |
| --- | --- |
| 新建采集 | `shadow_batch` runner |
| 资料来源 | `urls_*.txt` 文件 |
| 字段规则 | `schema/normalizer` + `parser/css_parser` |
| 采集结果 | `acs_shadow_logs/acs_shadow.jsonl` |
| 导出报告 | `storage/export_json` + CSV 生成 |
| 高级设置 | `health_check` / `readiness` / `on_mode_readiness` / `release_checklist` |

> 所有模块接口不变，仅 UI 层封装。

---

## 13. 安全机制不变声明

| 机制 | 状态 |
| --- | --- |
| ACS_MODE=shadow | **不变** |
| AI Parser 不替代旧流程 | **不变** |
| Self-Healing 不自动覆盖 | **不变** |
| canary 不执行于真实平台 | **不变** |
| 403/401 不绕过 | **不变** |
| API Key 不泄露 | **不变** |

> 仅改 UI 层的展示和用户流程。底层所有合规/安全/测试/审计模块保留。
