# ACS 资料采集助手 — 桌面使用说明

## 软件定位

**ACS 资料采集助手**：输入主题或导入资料来源，自动发现候选资料，用户确认后测试采集、预览并导出为 JSON / CSV / Markdown。

---

## 环境要求

- Windows 10/11
- Python 3.10+
- 不需要联网（本地运行）
- 不需要管理员权限

---

## 依赖安装（仅首次）

打开终端，进入项目目录，运行：

```bash
python -m pip install -r requirements.txt
```

所需依赖：`flask` `beautifulsoup4` `lxml` `requests`

> 注意：启动器不会自动联网安装依赖。如网络不可用，请提前手动安装以上 4 个包。

---

## 如何启动

```
双击 start_acs_desktop.bat
```

启动后依次执行：

1. 检查 Python + 依赖
2. 检查端口 5020 是否空闲
3. 启动本地服务 `127.0.0.1:5020`
4. 等待服务健康检查通过
5. 自动打开浏览器 → ACS 资料采集助手界面
6. 首页显示"✅ 本地服务已连接"

---

## 如何使用

### 主流程

```
1. 智能找资料 → 选择发现方式 → 输入主题/关键词 → 开始找资料
2. 候选确认 → 勾选需要的条目（blocked 禁止勾选）→ 加入采集任务
3. 采集结果 → 开始测试采集 → 进度监控 → 结果表格
4. 导出资料 → JSON / CSV / Markdown
```

### 导航说明

| 导航 | 功能 |
| --- | --- |
| 首页 | 服务状态 + 快捷入口 |
| 智能找资料 | 输入主题 → 自动发现候选来源 |
| 新建采集 | 创建采集任务 |
| 资料来源 | 导入 URL 文件 / 本地文件 |
| 字段规则 | 选择要提取的字段 |
| 采集结果 | 进度监控 + 结果表格 + 导出 |
| 导出报告 | 选择导出格式 |
| 高级设置 | 技术诊断（安全/合规/就绪度） |

---

## 如何关闭

在启动窗口按 `Ctrl+C`，服务自动停止。

（如果双击 bat 的窗口已关闭，重新打开一个终端，找到 `python -m acs.web.local_server` 进程并关闭）

---

## 常见问题

### 端口 5020 被占用

```
[错误] 端口 5020 已被占用。

解决: 关闭占用 5020 端口的其他程序。
查看占用进程: netstat -ano | findstr :5020
```

### 依赖未安装

```
[错误] 未安装 Flask 或其他依赖。

解决: pip install -r requirements.txt
```

### 服务未连接

```
首页显示"⚠️ 本地服务未连接"

解决: 双击 start_acs_desktop.bat 启动服务
      确保启动窗口未关闭
```

### 页面打不开

```
检查: start_acs_desktop.bat 窗口是否还在运行
检查: 浏览器是否拦截了本地文件
```

### 路径含中文

```
当前项目已支持中文路径。bat 文件首行已设置 chcp 65001。
如仍有问题，请将项目移到不含中文的路径。
```
