# 安全策略

## 报告安全漏洞

如发现安全漏洞，请通过以下方式报告：

- **私密报告**：通过 GitHub Security Advisory（如仓库已公开）或直接联系维护者
- **响应时间**：我们会在 72 小时内确认收到报告，并在 30 天内提供修复方案
- **请勿公开披露**：在修复发布前，请勿在公开渠道讨论漏洞细节

## 安全设计原则

### API Key 保护
- API Key 使用 Windows DPAPI（Data Protection API）按当前系统用户加密存储
- 配置文件（`ai_settings.json`）中不保存明文 Key
- 日志输出中 Key 自动脱敏（`mask_api_key()`）
- 任务档案仅保存 Key 摘要，不保存明文

### 网络访问
- 所有网络请求仅访问用户显式输入的 URL
- AI API 调用使用用户自行配置的 endpoint 和 Key
- 无不透明的后台数据上传
- Firecrawl 远程服务需用户主动配置和启用

### 文件系统
- 运行数据保存在 `%LOCALAPPDATA%\UniversalWebCollector`
- 导出文件保存在当前工作目录的 `采集结果导出\` 下
- 不扫描或修改用户文件系统（采集范围仅限用户输入的 URL）
- 临时文件仅限于自测运行时目录（`self_test_runtime\`）

### 浏览器自动化
- Playwright 使用独立的浏览器 profile，不访问用户日常浏览器数据
- 浏览器自动化默认启用但可通过复选框关闭
- 登录态（Cookie/LocalStorage）存储在 `chrome-profile\` 中，已加入 `.gitignore`

### 用户数据
- 采集结果存储在本地 SQLite 数据库，不上传云端
- 变更提醒和计划采集配置存储在本地 JSON 文件中
- 用户可通过导出功能将数据保存为 Excel/CSV/JSON 自行管理
- 软件卸载后，用户数据保留在 `%LOCALAPPDATA%` 中，用户可手动删除
